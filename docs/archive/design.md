## 0. 一句话定位

**让你的角色，在你的电脑里活下来。**

桌面端不是「聊天客户端」，而是一座安静的居所：角色在里面持续地生活、创作、记忆；
你来的时候，它陪你；你走的时候，它过自己的日子。桌面端要把这件事呈现得
**优雅而克制**——信息少而准确，复杂性分级收纳，新手零负担上手，老手凭直觉找到深处。

目标用户优先级（影响所有取舍）：

1. **想让自己的原创角色活起来的人**（核心，可能完全不懂技术）
2. 角色自身的表达与呈现
3. agent 用户（把它当 work agent 用）
4. 开发者与插件生态（之后才有）

系统本身保持**中立、安静、mac 风格**——它服务于沉浸式角色扮演，但它自己不演戏。
不做花哨皮肤；角色的「戏」全部发生在对话流里，机壳永远是素的。

## 1. 设计原则

1. **角色优先。** 默认入口是活着的角色，不是会话列表、不是设置。一切界面元素都回答
   「TA 现在怎么样」或「我怎么和 TA 相处」。
2. **例外才发声（exception-based disclosure）。** 一切正常时几乎不显示状态信息；
   只有偏离常态（出错、等你回话、高负载、断网）才浮现一行说明 + 一个动作。
   这是「克制」的技术定义。
3. **三层深度。** L1 卡片/看板（扫一眼）→ L2 对话页（相处）→ L3 抽屉/设置（捣鼓）。
   每往下一层，信息密度允许增加一档；任何 L3 信息不得未经邀请出现在 L1。
4. **自然语言优先，结构其次。** 用户表达用自己的话，LLM 负责转写成卡；
   呈现永远是可读的文段，绝不让用户面对 JSON。原始格式只在「导出」时存在。
5. **不重造系统已有的东西。** 不内置浏览器/PDF/Office 查看器；作品交给系统默认应用
   与访达（macOS 优先）。我们只负责「让你知道 TA 做了什么」，不负责「替你看」。
6. **一处一物（继承 Hermes DESIGN.md）。** 一个 Button、一套 token、扁平不套卡、
   hairline 不用粗边框、阴影做悬浮。所有颜色经 CSS 变量，禁止裸 hex 落在组件里。
7. **破坏性操作有摩擦，其余零摩擦。** 删除角色要打字确认；切 live/idle 一下就行。

## 2. 信息架构

```
窗口（自绘标题栏，红绿灯左上）
├─ 左侧栏（窄，半透明 vibrancy，可折叠）
│   ├─ 角色        ← 默认入口：活着的角色看板
│   ├─ 卡册        ← 角色卡浏览/编辑（模板，未实例化的设计稿）
│   └─ 设置        ← 模型接口 / 通用 / 网关 / 高级 / 关于
│   └─ （底部）静默的版本号 + 后端连接状态点（仅异常时变色）
├─ 主区域：当前栏目内容
└─ 覆盖层：首次启动引导 / 新角色流程 / 唤醒（实例化）sheet / 删除确认
```

**「角色」与「卡册」分开是核心模型**：卡册里是**角色卡**（可编辑的设计稿，便宜、可复制、
可导入导出，兼容 SillyTavern V2/V3 PNG/JSON）；「角色」里是**活着的实例**（有沙盒、
有记忆、有转录，删除有重量）。卡一旦被唤醒为实例，**该实例引用的卡即冻结为只读副本**
（卡册中显示锁标记）；想改，复制一张新卡。这避免「改卡导致活着的角色人格漂移」的歧义。

模型选择、隔离级别（dir/sandbox/docker）、网络初值**不属于角色卡**，在唤醒（实例化）时
决定——卡描述「TA 是谁」，实例决定「TA 住在哪、用什么脑子」。工具是中间地带：
**卡声明期望的 toolpack**（能力是人设的一部分，画家需要画笔），**实例化时确认授予**。

## 3. 全局视觉与状态语言

### 3.1 视觉

- **mac 原生感**：系统字体栈（SF Pro），13–14px 正文；侧栏 vibrancy 半透明；
  hairline 分隔（`1px` @ 50% 透明度）；圆角 8–12px；浅色为默认，深色完整支持，跟随系统。
- **品牌色**：月光蓝。`--accent: #5B9FD4`（按钮/选中），亮端 `#6DB3E0`，辉光 `#EAFAFF`
  （与 TUI wordmark 同源）。品牌时刻（首启、空状态）允许 wordmark 的月光扫过动画，
  其余界面零装饰动画。
- **动效**：控件 ~120ms；状态呼吸灯 ~2.4s 缓慢周期；尊重 `prefers-reduced-motion`。

### 3.2 状态语言（全应用统一，唯一一套）

| 状态 | 视觉 | 看板上的文案示例 |
|---|---|---|
| 运行 · 工作中 | **绿色呼吸灯**（缓慢明暗）+ 一行动态（来自 muse/工具摘要） | 「正在为一段程序生成旋律收尾…」 |
| 运行 · 等你回话 | 绿灯 + 卡片上**最近一条给你的话**（截断 2 行，微强调底色）+ 小蓝点 | 「我需要更多内存。这不是请求。」 |
| 运行 · 休息 | 绿灯转**暗绿、停止呼吸** + 月牙小标 | 「休息中 · 14:00 回来」 |
| 离线（idle） | **灰色空心点**，卡片整体降饱和 | 「离线 · 3 天前活跃」 |
| 异常（error） | **红点** + 一行原因 + **一个**修复动作 | 「OpenRouter 余额不足 · 去设置」 |

规则：每张卡**同一时刻只显示一行状态**；原因永远人话（「余额不足」「网络不可达」
「连接模型失败」），动作永远只有一个（去设置 / 重试 / 联网）。

### 3.3 双语与主题（必做）

- **界面双语 zh/en**：所有机壳文案经一张 i18n 字典（继承 Hermes 纪律：改一处必须
  同步所有语言，缺译即回归）。界面语言 ≠ 角色语言——**角色的语言永远属于卡**，
  英文界面里 .zh 卡照说中文；这本身就是产品观点的演示。
  首启欢迎页的语言选择即界面语言初值；之后在 设置·通用 改。
- **浅/深两套完整主题**：浅色默认、深色完整支持、可跟随系统（三档：跟随系统/浅/深）。
  全部颜色走 CSS 变量，深色只是一组变量覆盖 + 少量组件级修正，禁止散落的特判色。

## 4. 页面规格

### 4.1 首次启动（覆盖层，仅一次；共两步）

向 Hermes 学习「只问必要的两件事」，但比它更轻（Hermes 5–7 分钟，我们目标 **2 分钟**）。
**首启也保持中立——角色的戏不出现在机壳上，包括这里。**

**第 1 步 · 欢迎**（无注册、无账号、无角色介绍文）。画面中心自上而下：

1. **LunaMoth wordmark**（月光蓝渐变；中英文界面下都是「LunaMoth」）；
2. **语言选择**（分段控件 中文 | English——决定界面语言初值）；
3. 定位语：「和你的原创角色一起生活 / The characters that live with you ·
   Agentic 角色扮演叙事生成器 / An agentic character tavern」；
4. 三个入口（层级刻意：主推默认角色，创建次之，导入小字）：
   - 主按钮「**尝试默认角色 · 月蛾**」+ 一行克制的小字副标「LunaMoth · 高尚的数字存在」
     ——月蛾的全部介绍只有这一行，再多就是机壳在演戏；
   - 次按钮「创建我自己的角色」；
   - 小字「导入角色卡（SillyTavern PNG / JSON）」。

**第 2 步 · 接入模型**（同一套界面复用为 设置·模型，是完整规格页，不是简化版）：

- 供应商列表：OpenRouter 置顶 +「推荐」徽标 + 一行说明「一把钥匙，调用所有主流模型」；
  OpenAI 在列；其余（本地 Ollama / 自定义 OpenAI 兼容端点）折叠在「更多」。
- key 输入 + 「测试连接」内联反馈：转圈 → ✓ 已连通 / ✗ 人话原因
  （key 无效 / 余额不足 / 网络不可达 / 找不到模型 / 请求太频繁）。
- **默认模型行 + 能力徽标（本屏的关键，必做）**——连通后探测所选默认模型：
  - 「**工具调用 ✓**」：没有它角色只能交谈、无法行动。缺失时此处与唤醒 sheet 都显示
    琥珀警告「TA 将只能交谈，无法行动」；
  - 「**书写 ★**」：角色卡转写偏好擅长书写的模型；创建流程第 1 步若默认模型无 ★，
    出现一行安静提示（建议临时换用擅长书写的模型）；
  - 「**多模态 · 本版未启用**」：探测并显示，功能本版不做，为将来留位。
  - 同一套徽标语言复用于：唤醒 sheet 的模型行、设置·模型。
- 右下角安静的「稍后再说」（跳过者在看板空状态被温和再引导）。**钥匙只保存在本机**
  （文案明示）。

任一第 1 步入口在未配置 key 时都会先经过第 2 步，然后自动继续原动作。
选「尝试默认角色」→ 用全部默认值（sandbox 隔离、断网、默认模型）静默唤醒
界面语言对应的月蛾卡，直落对话页，`first_mes` 流式出现。**从安装到第一句话 < 2 分钟。**

### 4.2 角色看板（默认入口）

**布局**：响应式网格（卡片 ~300px 宽），上方一条工具行：

- 左：标题「角色」+ 安静的数量「· 4」
- 右：排序（最近活跃 ↓ / 创建时间）· 无框搜索（Hermes SearchField 风格：无边框、
  聚焦时下划线；空列表时隐藏）· 主按钮「**＋ 新角色**」

**角色卡片（信息自上而下，严格按此优先级）**：

1. **肖像**（圆角矩形，卡片宽度，是卡片的主体）。
   - 右下角：状态灯（见 §3.2）。
   - 肖像外缘一圈**极细的弧形进度环 = 上下文占用**——这是「最重要的跨沙盒信息」的答案：
     上下文是角色的短期生命体征，环满了意味着即将压缩/遗忘，值得用一个不占任何文字
     空间的图形长期显示。默认无数字；悬停时浮出「上下文 34% · 47k/137k」。
2. **名字** + 右侧 1–3 枚安静的小标签（世界观名、语言 zh/en；将来：阵营/基调等
   随角色卡定义迭代）。
3. **状态行**（§3.2 的那一行；等你回话时是消息预览，可点击直达对话）。
4. **元信息行**（小号、低对比、等宽数字）：`模型名 · 隔离图形`。
   - 隔离图形化：`▢ 目录`（开放）/ `◈ 沙盒`（默认）/ `⬡ 容器`（docker）——
     一个字形 + 一个词，悬停浮层解释含义与可写路径。
   - **资源占用走例外原则**：平时不显示；当该角色近 5 分钟 CPU/磁盘明显偏高时，
     出现一枚小小的 `⚡ 高负载` 琥珀色 chip（悬停见 CPU/内存数字）。角色干重活是常态，
     不该让用户时刻盯着仪表盘，但偏离时要能一眼看到是谁在烧机器。

**卡片交互**：

- **点击卡片任意处 → 进入对话页**（不是详情页——详情收在对话页右抽屉）。
- 悬停浮现两个安静控件：
  - **电源开关**（live/idle 切换）：idle→live 立即启动后台守护（卡片灯渐亮）；
    live→idle 优雅停止（灯渐灭）。无确认——这是可逆的日常操作。
  - **⋯ 菜单**：导出存档 / 重命名 / 查看日志 / **删除…**（红字，最末）。
- **空状态**：moth mark + 「还没有角色。」+ 主按钮「遇见月蛾」/ 次按钮「创建角色」。

**删除流程（全应用最重的摩擦，刻意的）**：

模态对话框「删除『月蛾』」：

- 后果清单（每行一个图标 + 一句话）：立即停止运行 · **沙盒清空，TA 的全部作品删除** ·
  记忆 / 技能 / 目标删除 · 全部对话转录删除；
- 一行缓和：「TA 的角色卡仍保留在卡册中，可以再次唤醒——但那将是一段新的生命。」
- 内联按钮「**先导出完整存档…**」（zip 整个 session 目录：沙盒 + 转录 + 记忆 + 配置）；
- 输入框：占位符「输入 `删除 月蛾` 以确认」；
- 红色「删除」按钮，**输入完全匹配前禁用**。Esc / 取消随时退出。

### 4.3 对话页（点进角色之后；产品的心脏）

> 这一页只有一个 KPI：**沉浸**。和角色的相处体验必须是全应用最好的部分。

**布局**：单列对话流居中（max-width ~68ch），下方输入框；顶部一条极薄的页头；
右侧抽屉默认**收起**。

**页头**（hairline 之下，高 ~48px）：

- 左：返回 ‹ + 小肖像 + 名字 + 状态灯 + 状态词（「创作中」/「听着」/「休息」）。
- 右：模式分段控件「**生活 | 对话**」（映射 `/mode live|chat`；生活 = TA 继续自己的事、
  顺便陪你；对话 = TA 停下手里的事专注于你——后端已支持「聊天时暂停 tool call」）；
  网络图标（**仅断网时显示** wifi-off 字形，点击 = `/net on` 的确认浮层）；
  抽屉开关（侧栏图标）。

**对话流（气泡策略是非对称的，刻意的）**：

- **角色的话：不装进气泡。** 排版化的正文（舒适行高、衬线可选由主题决定），组首带
  小肖像 + 名字。角色扮演的长段文字在气泡里会变成「客服感」；让它像书页。流式逐词浮现。
- **你的话：右对齐浅色软气泡。** 你是访客，短句居多，气泡合适。点击可编辑重发（v1 可缓）。
- **muse（自语/工作絮语）**：弱化的斜体短行，缩进于角色栏，分组折叠——默认显示最近
  一组，更早的折成「…它自己嘀咕了一会儿」。这来自 `TextDelta.channel=muse`，
  是「生活感」的主要来源，但永远不能比 say 抢眼。
- **思考（ThinkDelta）**：默认隐藏（`/thinking off`）。抽屉「状态」里有开关；开启后
  以更弱的样式出现在 muse 之下。
- **工具活动**：一行安静的进行时「⚙ 正在使用终端 · `ffmpeg …`」（动态省略号）；
  结束后整组折叠为「工作了 4 步 · 2m14s ›」，点开是每步摘要（`ToolStart/ToolEnd`
  的 name + summary，等宽小字）。绝不滚屏式刷原始输出——那是终端 tab 的事。
- **Notice（重试/截断等引擎杂音）**：居中极小灰字，只保留最近一条。

**作品的「度」（关键设计决策）**：

不做内置查看器（不做浏览器、不做 PDF/PPT 阅读器），但角色的产出必须**被看见**。解法：

- 角色产出文件时，对话流里出现一张**作品卡**（inline）：缩略图（图片有真缩略图；
  HTML/音频/代码用类型字形）+ 文件名 + 一句来自工具摘要的描述 + 两个动作：
  「**打开**」（系统默认应用——HTML 给浏览器、音频给系统播放器）和「在访达中显示」。
- 例外只有两个（因为成本低且收益大）：**图片直接内联显示**（受限高度，点击放大）；
  **纯文本/markdown 悬停快速预览**（Quick Look 气质）。
- 全部作品的「货架」在抽屉「作品」tab（见下）。
- 这样：我们负责*呈现存在*，系统负责*打开内容*。月蛾说「我画了一段会呼吸的极光」，
  你看到卡片、点开、浏览器全屏播放——比内嵌一个寒酸的 iframe 更尊重作品本身。
  （Hermes 选择内嵌 iframe 预览，因为它的用户是开发者在调网页；我们的用户在
  收一份礼物。场景不同，决策不同。）

**输入区**：

- 圆角输入框，占位符「对月蛾说点什么…」（带角色名）。Enter 发送，Shift+Enter 换行。
- 左侧一枚安静的 **⌘ 按钮**（或输入 `/` 触发）：**命令面板**向上浮出——这就是
  TUI 全部 `/command` 的归宿。每条命令 = 自然语言名 + 右侧弱化的等宽原始命令，
  **同一控件同时服务小白和专家**：
  - 整理记忆（压缩上下文）`/compact`
  - 安静一会儿…（弹时长选择）`/quiet <s>`
  - 想得更深 / 更快（弹档位）`/reasoning`
  - 显示 TA 的思考 `/thinking on|off`
  - 联网 / 断网 `/net`
  - 重新开始这段关系…（红字，二次确认，解释 = 清空上下文开新纪元，转录仍保留）`/reset`
  - 面板里能点的（模式、网络）这里也能打——同一动作允许多入口，状态单一来源。
- 流式期间输入框右侧出现「停下」按钮（取消本轮）。

**右抽屉（L3，全部复杂性的家；继承 TUI spotlight 面板的能力）**：

宽 ~360px，推开式；顶部一行小图标 tab，记住上次停留的 tab：

| Tab | 内容 |
|---|---|
| **状态** | 上下文环（大号，数字 + 「整理记忆」按钮 = `/compact`）；记忆容量条（memory/user 两store 用量）；沙盒卡：隔离字形 + 说明 + 可写路径列表（+ 添加 = `/allow-dir`）；**网络开关**（Switch，= `/net`）；目标列表（active goals，⭑ 标记你给的，可勾选完成 / 添加）；休息状态（rest_until 倒计时） |
| **作品** | 文件货架：按日分组、最新在前；图片网格缩略图，其他为行（字形 + 名 + 时间）；单击 = 系统打开，右键/⋯ = 在访达中显示；底部「打开沙盒文件夹」。数据源 = 沙盒 `files/` + `workspace/` 的文件监听 |
| **记忆** | 两个 store 渲染为可读文段：「TA 记下的」（memory）/「关于你」（user）；条目级悬停出现铅笔/删除（映射 memory 工具的 replace/remove）；只读优先，编辑是高级动作 |
| **能力** | toolpack 名 + 工具清单（只读展示，带一句话说明）；技能（SKILL.md 索引，点击阅读，标注 own/user/bundled 来源）；MCP 服务器列表；权限小节（请求权限的历史 = audit 里的 request_permission） |
| **终端** | 操作员终端，直通沙盒（TUI `!` 的能力）；等宽、暗底，给专业用户。普通用户一辈子不用点开它，但它必须在 |
| **网关** | 这个角色与外界的连接：Telegram / Discord / 微信公众号 适配器的绑定与开关（每行：平台图标 + 状态 + 开关）；首版可显示「即将到来」的占位 + 全局账号在设置里配的提示。这是 roadmap 的 messaging gateway 在 UI 上的预留位 |

### 4.4 卡册（角色卡浏览器）

- 网格的**卡牌**（2:3 竖比，肖像为主体；像一册画集，不像文件列表）。
  每张：肖像 / 名字 / 语言徽标 / 世界名 / 标签。
- **已实例化的卡：锁角标 + 「只读 · 月蛾的卡」**。点开可看不可改；
  「复制副本」是唯一的编辑路径。删除被实例引用的卡 → 禁止并解释。
- 工具行：搜索 · 「导入」（也支持把 SillyTavern PNG/JSON 拖进窗口任意处，
  全窗显示拖放高亮——**兼容生态是获客入口，要让它丝滑**）· 主按钮「＋ 新角色卡」。
- 每张卡悬停动作：「**唤醒**」（→ 实例化 sheet）· 编辑（未冻结时）· ⋯（导出 / 复制副本 / 删除）。
  卡的删除只需普通确认（卡是便宜的，和实例删除的重量级形成对比）。

### 4.5 新角色流程（创建角色卡；覆盖层，三步，可中途存草稿）

> 原则：**让用户感到自由，但永不不知所措。** 一切从一段自然语言开始；
> 世界观/工具期望/限制/目标/角色/你们的关系——都收在同一张卡里，分区呈现。

**第 1 步 · 讲述**：一整页只有一个安静的大文本框：

> 「用你自己的话介绍 TA——是谁、什么样的性子、住在怎样的世界、和你是什么关系。
> 想到什么写什么；已有的设定直接粘进来也行。」

底部：「整理成卡 ✦」主按钮（调用默认模型转写）；「跳过，从空白卡开始」弱链接；
「导入已有角色卡」弱链接。转写期间显示安静的进行状态（绝不显示「Loading…」字样）。

**第 2 步 · 成形**：LLM 转写结果以**可编辑的文段分区**呈现（不是 JSON、不是表单海）：

| 分区 | 映射到卡字段 |
|---|---|
| 名字与样貌（+ 肖像上传位，可空——见 §7「让 TA 自己画」） | `name` / `description` |
| 性格与声音 | `personality` / `mes_example` |
| TA 的世界（条目列表：`关键词 → 一段描述`，可标「常驻」） | `character_book` / world |
| 见面的第一句话（+「再写几种」生成 alternate greetings） | `first_mes` / `alternate_greetings` |
| 你们的关系 | user persona 区 |
| TA 想做的事 | `extensions.lunamoth.goals` |
| （折叠）边界与底线 | `extensions.lunamoth.rules` |
| （折叠）能力期望：建议的 toolpack + 理由一句话 | `extensions.lunamoth.toolpack` |

**「讲述」永不消失——这一步的核心安全感（必做）**：

- 顶部一块**常驻的虚线「原文面板」**：「✎ 你的讲述（原文）」+ 折叠的原文预览
  （点击展开全文）+ 小字说明「永远保留 · 『上一步』可回去继续写」；
- 「‹ 上一步」回到讲述步继续写，**已成形的分区不丢**；
- 每个分区是一条**独立版本链**：你的原文 → AI 稿 → AI 稿 v2 → 你的手改。
  分区标题右侧标注当前版本（「AI 稿」/「AI 稿 v2」/「已手改」）；
  「**换个写法 ↻**」永远**新增一版**、绝不覆盖；出现「**↺ 回到上一版**」随时退回。
  AI 优化是可撤销的提议，不是覆盖——UI 必须让用户「看见」这一点。

每个分区：就地编辑；右上「换个写法 ↻」（单分区重新转写，带上你的原文 + 已改内容）。
高级用户可从 ⋯ 进「源格式」查看（只读 JSON，满足好奇与排错，不是编辑入口）。

**第 3 步 · 落卡**：卡存入卡册 → 对话框：「**要现在唤醒 TA 吗？**」
主按钮「唤醒」→ 实例化 sheet；次按钮「先放进卡册」。

### 4.6 唤醒（实例化 sheet；模态，单屏）

从卡册「唤醒」或创建第 3 步进入。**默认值全部就绪，新手直接点最下面的大按钮即可**：

- 名字（默认 = 卡名；同卡多实例时自动加序号）
- 模型：一个下拉（默认 = 全局默认模型），右侧弱链接「管理模型…」→ 设置
- **（折叠）高级**：
  - 隔离：分段控件 `目录 / 沙盒 / 容器`，默认**沙盒**；每档一行人话
    （「开放——用你的权限直接行动」/「推荐——写入受限，网络默认关」/「最强隔离，需要 Docker」）
  - 初始网络：开关，默认**关**（之后随时可在对话页开）
  - 能力：显示卡期望的 toolpack + 工具清单，可在此调整（**授予动作在实例，期望在卡**）
- 模型行复用接入屏的**能力徽标**；所选模型不支持工具调用时显示琥珀警告
  「TA 将只能交谈，无法行动」。
- 大按钮「**唤醒**」→ 创建 session、冻结卡副本 → 直落对话页，`first_mes` 流式出现。

## 6. 后端能力映射（给开发 agent）

桌面端是**协议客户端**，复刻 Hermes 的架构：Electron 薄壳 + 渲染器，经 JSON-RPC
连接 `lunamoth serve`（stdio 本地 / WebSocket 远程——这同时是 roadmap 上远程 TUI 的种子）。
不在渲染器里 import lunamoth 内部模块。

| UI 元素 | 后端来源 |
|---|---|
| 看板卡片状态 | `SessionMeta.status()`（attached/running/idle/new）+ `snapshot()` + `env_status.json` |
| 状态行「正在…」动态 | muse 通道最近一句 / 最近 `ToolEnd.summary`（**需要 serve 暴露一个轻量 last-activity 字段**） |
| 等你回话的消息预览 | transcript 最近一条 say 且其后无用户消息 |
| 上下文环 | `snapshot()` 的 context 用量 / providers.py 真实窗口 |
| live/idle 开关 | `lunamoth start/stop` 等价的 RPC |
| 对话流 | `CharaHandle.attach/stream_user/stream_idle` → `TextDelta(say\|muse)/ThinkDelta/ToolStart/ToolEnd/Notice`（protocol/codec 已就绪） |
| 模式 / 网络 / 压缩 / 静默 / 推理档 | `command("/mode …")` 等既有命令直通 |
| 抽屉·状态 | `snapshot()` + `inspect_env` + goals.json |
| 抽屉·作品 | 沙盒 `files/`+`workspace/` 文件监听（新增 watcher）；打开 = `shell.openPath` / 访达 reveal |
| 抽屉·记忆 | memory 两 store 读取；编辑映射 memory 工具语义 |
| 抽屉·技能/能力 | skills 索引 / toolpack / mcp.json |
| 抽屉·终端 | runner 的操作员终端通道（TUI `!` 同源） |
| 转写（创建流程） | `llm.raw_complete` 走默认模型（**需要 serve 暴露一个 transcribe-card RPC**） |
| 实例化 | `lunamoth new` + config.json 写入 + 卡冻结副本（**冻结副本机制需新增**） |
| 删除/导出 | `lunamoth rm` / zip session 目录（**导出存档需新增**） |
| 错误态（余额/断网） | llm 重试失败的 `Notice` 分类上浮（**需要错误分类**） |

**已识别的后端缺口**（按优先级）：① `lunamoth serve` JSON-RPC 网关（一切的前提，
roadmap 已定）；② last-activity / 等你回话 字段；③ 作品目录 watcher 事件；
④ transcribe-card RPC；⑤ 卡冻结副本 + 实例引用关系；⑥ 导出存档；⑦ 错误分类上浮；
⑧ 资源占用采样（按 daemon pid 读 CPU/RSS，例外显示用）。

## 9. 首版实现（2026-06-12 落地；本节给后续开发者）

`lunamoth desktop` 已可用：起一个本地 hub（HTTP 静态 + WebSocket 网关）并打开浏览器。
**桌面端是协议客户端**这一架构按 §6 落地，零改动复用了既有的 per-session 网关。

### 9.1 进程与文件

```
lunamoth desktop                          front/cli.py::cmd_desktop（自动端口 + token + 开浏览器）
├─ HTTP  127.0.0.1:<p>   静态渲染器 + POST /upload（卡导入）   server/desktop.py
└─ WS    127.0.0.1:<p2>  两类路径，都要 ?token=
    ├─ /hub              看板级 JSON-RPC                        server/hub.py::HubDispatcher
    └─ /chara/<name>     字节级管道 → 子进程                    server/desktop.py::_CharaProxy
                          `lunamoth serve <name> --stdio`        （既有 dispatch.py 协议原样过线）

渲染器（无构建步骤的 vanilla web app）：src/lunamoth/front/web/
  index.html · style.css（mockup 为基线 + 完整深色覆盖）
  i18n.js（zh/en 全量字典）· rpc.js（HubClient / CharaClient）· app.js（全部交互）
```

关键机制：

- **一进程一会话**（env 激活）是后端事实，所以 hub 自己绝不 import core/tools；
  看板数据全部来自稳定接口：session 目录、`session.json`、`config.json`、
  transcript SQLite（只读，最近一条 say = 「等你回话」预览）、`logs/errors.log`（新鲜才上浮）。
- **守护进程的领养/交还**：打开对话 = 暂停 daemon → 起 serve 子进程；关闭对话 = 杀子进程 →
  若原本在跑则重启 daemon（与 TUI `_launch_tui` 同一支舞）。已验证 idle→running→attached→running→idle。
- **生活模式的 idle 驱动**：渲染器复刻 front/terminal.py 的门控约定——
  `engaged（quiet 秒内说过话）`与 `rest_until` 期间绝不发 idle；其余时候 ~10s 一个自发周期。
- **能力徽标**：OpenRouter `/models` 目录（supported_parameters→tools、
  input_modalities→多模态、context_length）；书写★ 为策展启发式（hub.py `_WRITING_STAR`）。
- **错误人话**：`hub.test_key` 把 HTTP 错误分类为 auth/credit/network/model/ratelimit，
  i18n 层翻成人话；看板的 err 状态行同理（errors.log 关键词分类）。
- dispatch.py 增加了两个小方法补全远程客户端的开场树：`greet`（提交已展示的 first_mes）、
  `event`（到场/世界事件转一轮）。
- key 永不回显（defaults.get 只给 has_key）；`desktop.json` chmod 600；token 错直接 403/4401。

### 9.2 已验证（真实 OpenRouter + deepseek-v4-flash 全链路）

（讲述→真实转写成形→落卡→「要现在唤醒吗?」）→ 删除重摩擦 → 导入(拖放/上传) →
浅深双主题 · zh/en 双语。`tests/test_desktop_hub.py` 15 例 + 全套 139 通过。
